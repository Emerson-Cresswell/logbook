================================================================================
1. CURRENT STATE SUMMARY (authoritative)
================================================================================

MyLogbook is a static local-first medical procedure/CPD logbook PWA.

Current architecture:
- HTML/CSS/JavaScript (no build tool)
- service worker + manifest
- localStorage data model
- manual JSON backup/import
- Excel export via xlsx.full.min.js
- Firebase Hosting + GitHub Actions workflows

Hosting/deployment workflow (current):
1. Create/update a feature branch.
2. Open/update PR against main.
3. GitHub Actions run syntax + Firebase PR preview deploy.
4. GitHub Actions also auto-deploy the PR branch to stable test site https://test.mylogbook.uk (target `test`, live channel).
5. Latest PR update overwrites https://test.mylogbook.uk.
6. This is acceptable because project workflow normally runs one active PR at a time.
7. Perform installed-PWA checks using https://test.mylogbook.uk before merge.
8. Merge to main only after checks pass and user approves.
9. Main merge/push deploys production live site https://app.mylogbook.uk (target `live`, live channel).
